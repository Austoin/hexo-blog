## 题目描述

**LeetCode 1545. 找出第 N 个二进制字符串中的第 K 位**

给定两个正整数 `n` 和 `k`，二进制字符串 `Sn` 的形成规则如下：

- `S1 = "0"`
- 当 `i > 1` 时，`Si = S(i-1) + "1" + reverse(invert(S(i-1)))`
- 其中 `+` 表示拼接，`reverse` 为反转，`invert` 为按位翻转 0 和 1

**目标**：求 `Sn` 的第 `k` 位字符。

如果按照规则强行构造完整字符串，当 `n` 较大时（字符串长度为 `2^n - 1`），会引发严重的内存溢出和超时。因此，我们需要寻找规律，通过**逆向推导**将时间复杂度降至 `O(n)`，空间复杂度降至 `O(1)`。

## 解题思路

解题的关键在于理解字符串生成的对称性。每次生成新的字符串，其实就是把旧字符串放在左边，中间补个 "1"，右边放旧字符串的处理版。

### 中间位置的值为什么永远是 '1'？

根据拼接规则 `S(i-1) + "1" + ...`，新增加的字符 "1" 刚好把左右两部分分隔开，所以它永远位于字符串的正中间。

我们列举前几项来看看：

- `S1 = "0"` (长度1，中间位置1，值为0，这是唯一特例)
- `S2 = "0" + "1" + "1"` (长度3，中间位置2，值为1)
- `S3 = "011" + "1" + "001"` (长度7，中间位置4，值为1)

**结论**：对于任意 `n > 1` 的字符串 `Sn`，其长度必定是 `2^n - 1`，中间位置 `mid` 必定是 `2^(n-1)`。并且该中间位的值**固定为 '1'**。

### 如果 k 不在中间怎么办？（位置映射）

已知字符串由 `左半部分 + 1 + 右半部分` 组成，左半部分其实就是上一层的 `S(n-1)`。

我们可以通过判断 `k` 和 `mid` 的大小关系，将问题逐步缩小到 `S(n-1)` 中：

**1. k < mid（在左半部分）**

因为左半部分就是 `S(n-1)` 本身，所以第 `k` 位的值与 `S(n-1)` 的第 `k` 位完全相同。无需任何处理，直接进入下一轮。

**2. k > mid（在右半部分）**

右半部分是 `S(n-1)` 经过**翻转(invert)**和**反转(reverse)**得到的。

- **反转(reverse)带来的位置映射**：右半部分的第 `k` 位，对应左半部分（也就是 `S(n-1)`）的哪个位置呢？根据对称性，映射公式为 `映射位置 = mid * 2 - k`
- **翻转(invert)带来的值变化**：因为经过了按位翻转，所以我们找出来的结果还需要取反一次。为了不破坏循环结构，我们可以定义一个累加器 `invertNum` 来记录**总共取反了多少次**

### 最终的收敛

随着 `n` 不断减小（每次减 1），最终会触发两种终结条件之一：

1. **中途撞到中间位**：某次循环中 `k` 刚好等于当时的 `mid`。因为中间位必定是 '1'，我们只需要根据累加的 `invertNum` 判断最终是 '1' 还是 '0'（偶数次取反不变，奇数次 1 变 0）
2. **一路缩圈到 n=1**：此时来到了基本情况 `S1 = "0"`。同理，根据 `invertNum` 的奇偶性判断 0 最终会变成什么

## 完整代码实现

```go
func findKthBit(n int, k int) byte {
    invertNum := 0
    for n > 1 {
        mid := int(math.Pow(2, float64(n-1)))
        
        if k == mid {
            if invertNum%2 == 0 {
                return '1'
            } else {
                return '0'
            }
        }
        
        if k > mid {
            k = mid*2 - k
            invertNum++
        }
        
        n--
    }
    
    if invertNum%2 == 0 {
        return '0'
    } else {
        return '1'
    }
}
```

## Go 语言语法避坑指南

在将这段逻辑从 Java/C++ 等语言转换为 Go 时，有几个新手极易踩坑的语法差异：

**坑点一：Go 语言不支持三元运算符**

在其他语言中常用的 `invertNum % 2 == 0 ? '0' : '1'` 会在 Go 中报语法错误（`invalid character '?'`）。Go 官方为了追求代码的清晰单一，刻意去掉了三元运算符，必须老老实实写成 `if-else` 结构。

**坑点二：编译器的严格 Return 检查**

如果你的循环条件写的是 `for n > 0`，并且在循环末尾没有任何 `return` 语句，Go 编译器会报错 `missing return`。因为编译器无法确定 `for` 循环内的 `if` 必定会被触发。

**最优解法**是将循环条件设为 `for n > 1`，然后在函数最外层收底返回 `n=1` 时的结果，这样不仅逻辑更符合业务（`n=1` 本身就不需要循环查找），也完全符合 Go 编译器的语法检查规则。

## 复杂度分析

- **时间复杂度**：`O(n)`，每次循环 `n` 减 1，最多循环 `n` 次
- **空间复杂度**：`O(1)`，仅使用常数个变量

通过逆向推导和位置映射，我们成功避免了构造完整字符串，将问题的时间和空间复杂度都降到了最优。
