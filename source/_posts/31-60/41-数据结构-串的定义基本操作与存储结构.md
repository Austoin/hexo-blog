## 引言

进入第四章以后，我们正式从一般线性表转向一种更特殊、也更贴近实际应用的数据结构：**串（String）**。

很多同学第一次学串时会觉得它“不过就是字符数组”，但考研真正喜欢考的不是这个表面，而是：

- 串和普通线性表到底有什么区别
- 空串、空格串、子串、主串这些概念怎么区分
- 串的基本操作都有哪些
- 串的顺序存储和链式存储各自有什么优缺点

本文会把 `4.1` 相关内容一次讲透，并把 PPT 里默认省略的一些前提也补全好，方便你后面无缝衔接模式匹配与 KMP。

---

## 一、串的定义与基本概念

### 1. 串的定义

串（String）是由**零个或多个字符组成的有限序列**，一般记为：

$$S = 'a_1a_2\cdots a_n' \quad (n \ge 0)$$

其中：

- `S` 是串名
- 单引号或双引号括起来的字符序列是串值
- 每个 `a_i` 可以是字母、数字、标点或其他字符
- 串中字符个数 `n` 称为**串长**

当 `n = 0` 时，这个串称为**空串**。

例如：

- `S = "HelloWorld!"`
- `T = "iPhone 11 Pro Max?"`

> 注意：不同语言对字符串字面量使用的引号不完全一致，但数据结构里讨论的是抽象概念，本质不受影响。

### 2. 子串、主串、位置

这几个概念必须区分清楚：

- **子串**：串中任意个**连续字符**组成的子序列
- **主串**：包含某个子串的串
- **字符在主串中的位置**：字符在串中的序号
- **子串在主串中的位置**：子串第一个字符在主串中的位置

例如对于 `T = "iPhone 11 Pro Max?"`：

- `"iPhone"`、`"Pro M"` 是 `T` 的子串
- `T` 是 `"iPhone"` 的主串
- 字符 `"1"` 第一次出现的位置是 `8`
- 子串 `"11 Pro"` 在 `T` 中的位置也是 `8`

### 3. 空串 VS 空格串

这是经典易错点。

- `""` 是**空串**，长度为 `0`
- `"   "` 是**空格串**，里面包含 3 个空格字符，长度为 `3`

空格也是字符，只要串里有字符，就不是空串。

### 4. 位序默认从 1 开始

在数据结构教材和考研题里，**串中字符的位序通常从 1 开始**，而不是从 0 开始。

这和 C 语言数组下标不同，后面做 `SubString`、`Index`、KMP 时一定要特别小心。

## 二、串和线性表的联系与区别

串本质上是一种**特殊的线性表**，因为：

- 数据元素之间同样是一对一的线性关系
- 可以顺序访问，也可以进行插入、删除、查找等运算

但它和一般线性表又不完全一样：

### 1. 数据对象不同

- 线性表中的元素可以是任意类型
- 串中的元素通常限定为**字符**

### 2. 操作对象不同

线性表常以“单个元素”为基本操作单位；而串在实际处理中，往往以**子串**为操作对象。

比如：

- 线性表更关心“删除第 3 个元素”
- 串更关心“查找某个单词”“替换某个片段”“截取某段子串”

### 3. 现实意义不同

对于字符来说，单个字符的意义通常不大，**多个字符组成的序列**才有现实语义。这也是串结构要单独拿出来研究的根本原因。

## 三、串的基本操作

假设：

- `S = "iPhone 11 Pro Max?"`
- `W = "Pro"`
- `T = ""`

常见基本操作如下：

- `StrAssign(&T, chars)`：赋值，把 `chars` 赋给 `T`
- `StrCopy(&T, S)`：复制，用 `S` 复制得到 `T`
- `StrEmpty(S)`：判空，若 `S` 为空串则返回 `TRUE`
- `StrLength(S)`：求串长
- `ClearString(&S)`：清空，把 `S` 置为空串
- `DestroyString(&S)`：销毁串并回收空间
- `Concat(&T, S1, S2)`：串联接，得到 `S1 + S2`
- `SubString(&Sub, S, pos, len)`：从 `S` 的第 `pos` 个字符开始截取长度为 `len` 的子串
- `Index(S, T)`：在主串 `S` 中查找模式串 `T` 第一次出现的位置，不存在则返回 `0`
- `StrCompare(S, T)`：比较两个串的大小关系

例如：

- `Concat(&T, S, W)` 后，`T = "iPhone 11 Pro Max?Pro"`
- `SubString(&T, S, 4, 6)` 后，`T = "one 11"`
- `Index(S, W)` 返回 `11`

### 补充：这些操作为什么会考

因为它们几乎涵盖了串的三类核心问题：

1. **构造与销毁**：赋值、复制、清空、销毁
2. **访问与判断**：判空、求长、比较、取子串
3. **组合与查找**：联接、模式匹配

后面的模式匹配算法，本质上就是 `Index(S, T)` 的高效实现。

## 四、串的比较规则与字符编码

### 1. 串的比较规则

串的比较采用**从前往后逐字符比较**的方式。

规则是：

1. 从第一个字符开始依次比较
2. 一旦出现不同字符，谁的字符更大，谁的串就更大
3. 如果短串是长串的前缀，那么长串更大
4. 只有所有对应字符都相同且长度也相同，两个串才相等

例如：

- `"abandon" < "aboard"`
- `"abstract" < "abstraction"`
- `"abstract" < "abstract "`
- `"academic" > "abuse"`
- `"academic" = "academic"`

这就是词典序（字典序）比较。

### 2. 为什么字符能比较大小

因为字符进入计算机后，最终都会被编码成二进制数。

- **字符集**：规定有哪些字符，例如 ASCII、Unicode
- **编码**：规定这些字符如何映射为二进制数，例如 UTF-8、UTF-16

可以把它理解成一个映射：

$$y = f(x)$$

- `x` 是字符
- `y` 是对应的二进制编码

### 3. 考研里的默认约定

在 408 数据结构题目里，通常**默认每个字符占 1B**，目的是聚焦算法和数据结构本身，而不是字符编码实现细节。

### 4. 乱码的本质

乱码不是“文件坏了”，而是**写入和读取使用了不同的编码规则**。

也就是说：

- 写文件时按照规则 `f` 编码
- 读文件时错误地按规则 `g` 解码

同一串二进制就会被解释成完全不同的字符。

## 五、串的存储结构

串的存储和线性表一样，主要分为**顺序存储**和**链式存储**。

### 1. 串的顺序存储

串顺序存储的核心思想很简单：

- 用一段**连续存储空间**存放字符
- 再配合长度信息或结束标记描述串的有效范围

#### （1）定长顺序存储

最常见的形式是静态数组：

```cpp
#define MAXLEN 255

typedef struct {
    char ch[MAXLEN + 1];
    int length;
} SString;
```

教材里常采用“**`ch[0]` 废弃不用，位序从 `1` 开始，另设 `length` 记录长度**”的写法。

这样做的好处是：

- 串的位序和教材公式完全一致
- 做 `SubString`、`Index`、KMP 时不容易混淆

#### （2）顺序存储的几种常见方案

PPT 里给了几种典型设计：

1. `ch[0]` 存长度，后面存字符
2. 不单独设长度变量，靠约定位置记录长度
3. 用 `\0` 表示串结束（C 风格字符串）
4. `ch[0]` 废弃，单独用 `length` 记录长度（教材方案）

其中第 4 种最适合数据结构教学，因为理论位序和实现位序对应最直观。

#### （3）顺序存储的优缺点

优点：

- 随机存取方便
- 按位序取字符快
- 实现简单

缺点：

- 若用定长数组，容量固定，可能溢出
- 插入、删除时可能要大量移动字符

### 2. 堆分配的动态顺序存储

如果不想把容量写死，可以在堆区动态申请空间：

- 优点：空间更灵活
- 缺点：需要手动申请和释放，管理更复杂

这本质上还是顺序存储，只是把静态数组换成了动态数组。

### 3. 串的链式存储

链式存储与链表一脉相承。

最朴素的做法是：

- 每个结点存放 1 个字符
- 结点之间用指针链接

但这种方式有一个明显问题：**存储密度低**。

因为：

- 字符通常只占 `1B`
- 指针往往要占 `4B` 或更多

于是经常改进为：

- 每个结点存放多个字符（块链结构）
- 用更少的指针管理更多字符

#### 链式存储的优缺点

优点：

- 插入、删除某段子串时更灵活
- 不要求整段连续空间

缺点：

- 存储密度低
- 不利于随机访问
- 实际做题里不如顺序存储常见

> 408 里如果没有特别强调，串的实现通常优先按顺序存储来理解。

## 六、顺序串的基本操作实现

下面用教材常见的顺序存储结构实现若干核心操作。

```cpp
#include <iostream>
using namespace std;

#define MAXLEN 255

typedef struct {
    char ch[MAXLEN + 1];
    int length;
} SString;

bool StrAssign(SString &T, const char *chars) {
    int len = 0;
    while (chars[len] != '\0') len++;
    if (len > MAXLEN) return false;

    T.length = len;
    for (int i = 1; i <= len; i++) T.ch[i] = chars[i - 1];
    return true;
}

bool StrEmpty(SString S) {
    return S.length == 0;
}

int StrLength(SString S) {
    return S.length;
}

void ClearString(SString &S) {
    S.length = 0;
}

int StrCompare(SString S, SString T) {
    int len = S.length < T.length ? S.length : T.length;
    for (int i = 1; i <= len; i++) {
        if (S.ch[i] != T.ch[i]) return S.ch[i] - T.ch[i];
    }
    return S.length - T.length;
}

bool SubString(SString &Sub, SString S, int pos, int len) {
    if (pos < 1 || pos > S.length || len < 0 || pos + len - 1 > S.length) {
        return false;
    }
    for (int i = 1; i <= len; i++) Sub.ch[i] = S.ch[pos + i - 1];
    Sub.length = len;
    return true;
}

bool Concat(SString &T, SString S1, SString S2) {
    if (S1.length + S2.length > MAXLEN) return false;
    for (int i = 1; i <= S1.length; i++) T.ch[i] = S1.ch[i];
    for (int i = 1; i <= S2.length; i++) T.ch[S1.length + i] = S2.ch[i];
    T.length = S1.length + S2.length;
    return true;
}
```

### 这些操作里最容易错的地方

- `SubString` 的 `pos` 从 `1` 开始，不是 `0`
- `len` 不能为负
- `pos + len - 1` 不能越界
- `Concat` 之前要检查总长度是否超过最大容量

这也是 PPT 里最后提到“**存储空间扩展怎么办**”的原因：当顺序串空间不够时，联接操作未必一定成功。

## 七、做题时最容易踩的坑

### 1. 空串和空格串混淆

- 空串长度是 `0`
- 空格串长度取决于空格字符个数

### 2. 位序和数组下标混淆

理论题常按位序 `1, 2, 3...` 讨论；编程语言数组却通常从 `0` 开始。做题时一定先确认到底采用哪套编号。

### 3. `\0` 结尾和 `length` 变量混淆

在 C 语言字符串里常用 `\0` 标志结束；在教材串结构里更常见的是**显式保存 `length`**。

这两种都是合法方案，但分析题时要看题目采用哪一种。

### 4. 串比较不是只看长度

比较串大小时，先按字符逐个比较，**不是谁长谁就大**。

## 总结

本节你需要真正掌握的是下面这几件事：

1. **串是特殊线性表**，但操作对象更强调子串
2. **空串、空格串、子串、主串、位置** 这些定义必须完全区分
3. **基本操作** 包括赋值、复制、判空、求长、清空、销毁、联接、求子串、定位、比较
4. **顺序存储** 是串最常见的实现，教材里常用 `ch[0]` 废弃、`length` 记录长度
5. **链式存储** 理论上可行，但存储密度较低，考试中更多是作为对比理解
6. **字符集和编码** 决定字符如何表示，乱码本质是编码规则不一致

只要这一节打牢，下一节的模式匹配你就不会再把“子串”和“模式串”“位置”和“数组下标”混成一团。
