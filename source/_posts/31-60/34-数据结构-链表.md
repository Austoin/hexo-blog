## 零、引言

在上一节中，我们学习了线性表的**顺序存储（顺序表）**，它的优点是可以"随机访问"，但缺点也很明显：插入和删除操作需要移动大量元素，且需要预先分配大片连续的内存空间。

为了解决这些痛点，**链表（链式存储）**应运而生。链表不要求物理空间连续，各个节点通过"指针"散落在内存的各个角落，彼此手拉手相连。

本节内容是考研数据结构**最核心的基石**，无论是大题还是选择题，链表的指针操作都是必考项。请务必亲自动手画图、写代码！

## 一、单链表 (Single Linked List)

### 1. 单链表的定义与代码实现

单链表的每个节点除了存放数据元素外，还包含一个指向下一个节点的指针。

**核心代码定义：**

```c
typedef struct LNode {
    ElemType data;        // 数据域
    struct LNode *next;   // 指针域
} LNode, *LinkList;       // LinkList 等价于 struct LNode *
```

> **王道Tips (命名技巧)：**
> - 当我们强调这是一个**单链表**时，使用 `LinkList L;` (可读性更强)。
> - 当我们强调这是一个**节点**时，使用 `LNode *p;`。

### 2. 带头结点 vs 不带头结点

- **不带头结点：** 写代码非常麻烦！对于第一个数据节点和后续节点的操作逻辑不同，需要大量的 `if` 判断。
- **带头结点（强烈推荐）：** 头结点不存放实际数据，只是为了统一插入和删除的操作逻辑。"用过都说好！"

**初始化代码（带头结点）：**

```c
bool InitList(LinkList &L) {
    L = (LNode *)malloc(sizeof(LNode)); // 分配一个头结点
    if (L == NULL) return false;        // 内存不足，分配失败
    L->next = NULL;                     // 头结点之后暂时还没有节点
    return true;
}
// 判断是否为空： L->next == NULL
```

### 3. 单链表的插入操作

#### ① 按位序插入 `ListInsert(&L, i, e)`

找到第 $i-1$ 个节点，将其指针修改，新节点插入其后。平均时间复杂度 $O(n)$。

#### ② 指定节点的后插操作 `InsertNextNode(LNode *p, ElemType e)`

```c
bool InsertNextNode(LNode *p, ElemType e) {
    if (p == NULL) return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL) return false;
    s->data = e;
    s->next = p->next; // ① 新节点连上后面的链
    p->next = s;       // ② 老节点连上新节点 (顺序绝对不能反!)
    return true;
} // 时间复杂度 O(1)
```

#### ③ 指定节点的前插操作 `InsertPriorNode(LNode *p, ElemType e)` (考研高频神技)

在没有头指针的情况下，如何在节点 `p` 前面插入？

- **偷天换日法：** 我们无法找到 `p` 的前驱，但我们可以把新节点 `s` 插入到 `p` 的**后面**，然后**交换 `p` 和 `s` 的数据域**！

```c
s->next = p->next;
p->next = s;
s->data = p->data; // 将p中原来的元素复制到s中
p->data = e;       // p中元素覆盖为新元素e
// 时间复杂度 O(1)！
```

### 4. 单链表的删除操作

#### ① 按位序删除 `ListDelete(&L, i, &e)`

找到第 $i-1$ 个节点，将其指针绕过第 $i$ 个节点，指向第 $i+1$ 个节点，然后释放第 $i$ 个节点。

#### ② 删除指定节点 `p` (偷天换日法 2.0)

同样，不知道前驱怎么办？把 `p` 后继节点的数据复制到 `p` 中，然后删除 `p` 的后继节点！

```c
LNode *q = p->next;
p->data = q->data;  // 和后继节点交换数据
p->next = q->next;  // 将q节点从链中"断开"
free(q);            // 释放后继节点的空间
// 注意边界坑：如果 p 是最后一个节点，此法失效，只能从头遍历找前驱！
```

### 5. 单链表的建立 (核心大招)

#### ① 尾插法建表

准备一个尾指针 `r` 始终指向当前链表的最后一个节点。每次新节点都插入到 `r` 后面，然后 `r = r->next`。**生成的链表顺序与输入顺序一致。**

#### ② 头插法建表 (极其重要！)

每次都把新节点插入到头结点之后。

- **王道考点：** 头插法的核心应用是**链表的逆置**！遍历一个链表，把节点挨个摘下来用头插法重新插入，链表就反转了。

## 二、双链表 (Double Linked List)

单链表无法逆向检索，双链表在此基础上增加了一个指向前驱的指针 `prior`。

**结构定义：**

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior, *next;
} DNode, *DLinklist;
```

**插入操作核心 (在p节点后插入s)：**

修改指针时**注意顺序**，防断链！

```c
s->next = p->next;
if (p->next != NULL) {  // 如果p不是最后一个节点
    p->next->prior = s;
}
s->prior = p;
p->next = s;
```

**删除操作核心 (删除p的后继节点q)：**

```c
p->next = q->next;
if (q->next != NULL) {  // q不是最后一个节点
    q->next->prior = p;
}
free(q);
```

## 三、循环链表 (Circular Linked List)

### 1. 循环单链表

表尾结点的 `next` 指针不再指向 `NULL`，而是**指向头结点** `L`。

- **判空条件：** `L->next == L`
- **优点：** 从表中任意一个节点出发都能找到其他所有节点。很多时候我们会放弃头指针，只设立一个**尾指针 `R`**。这样找到表尾只需 $O(1)$，找到表头也只需 $O(1)$ (`R->next` 即为头结点)。

### 2. 循环双链表

- 表头结点的 `prior` 指向**表尾结点**。
- 表尾结点的 `next` 指向**表头结点**。
- **判空条件：** `L->next == L && L->prior == L`

## 四、静态链表 (Static Linked List)

**概念：** 用**数组**来实现链表，节点在内存中分配一整片连续空间。每个节点包含数据 `data` 和游标 `next`（下一个元素的数组下标）。

**结构定义：**

```c
#define MaxSize 10
typedef struct {
    ElemType data;
    int next;   // 下一个元素的数组下标
} SLinkList[MaxSize];
```

- 0 号节点充当"头结点"。
- `next == -1` 表示已经到达表尾。
- **优点：** 增删操作不需要大量移动元素，只需修改游标。
- **适用场景：** 不支持指针的低级语言；或者操作系统中 FAT 文件系统的磁盘块分配。

## 五、顺序表 vs 链表 (大PK，到底爱谁？)

面对开放性问题："实现线性表时，用顺序表还是链表好？"请从以下几个维度回答，轻松拿满分：

1. **逻辑结构：** 都是线性表，属于线性结构。
2. **存储结构（物理结构）：**
   - **顺序表：** 连续存储。优点是支持随机访问，密度高；缺点是大片连续空间分配困难，扩容不便（动态分配也有复制成本）。
   - **链表：** 离散存储。优点是分配灵活，扩容方便；缺点是增加了指针空间开销，只能顺序查找。
3. **基本操作效率：**
   - **创：** 顺序表需预分配大空间；链表只需分配一个头结点。
   - **查：** 按位查找顺序表 $O(1)$，链表 $O(n)$。
   - **增/删：** 顺序表需移动大量元素 $O(n)$；链表找到位置后只需修改指针 $O(1)$。
4. **最终结论：** 表长难以预估、经常需要增删元素 —— **选链表**；表长稳定、经常需要查找检索 —— **选顺序表**。

## 六、课后大题硬核解析 (408历年真题)

### 真题 1：寻找单链表的倒数第 k 个位置上的节点 (2009年 408统考题)

**【题目背景】** 课后综合题第17题。已知带头结点的单链表，查找链表中倒数第 $k$ 个位置的节点，若成功输出 data 并返回 1，否则返回 0。要求尽量高效（遍历一次）。

**【解题思路：快慢双指针法】**

定义两个指针 `p` 和 `q`，初始都指向第一个数据节点。

1. 让 `p` 指针先往前走 $k$ 步。
2. 如果 `p` 还没走完 $k$ 步就到了 `NULL`，说明链表长度小于 $k$，查找失败，返回 0。
3. 随后，让 `p` 和 `q` **同步往后走**。
4. 当 `p` 走到 `NULL` 时，`q` 刚好指向倒数第 $k$ 个节点！

**【C语言实现】**

```c
int Search_k(LinkList list, int k) {
    LNode *p = list->next, *q = list->next;
    int count = 0;
    while (p != NULL) { // p先跑
        if (count < k) {
            count++;
        } else {
            q = q->next; // q随后同步跑
        }
        p = p->next;
    }
    if (count < k) return 0; // 长度不够k
    printf("%d", q->data);
    return 1;
}
```

- **复杂度分析：** 只遍历了一次链表，时间复杂度 $O(n)$，空间复杂度 $O(1)$。

### 真题 2：找出两个单链表的共同后缀起始位置 (2012年 408统考题)

**【题目背景】** 课后综合题第18题。假定两个单词用链表存储，如果有相同后缀，则共享后续的节点空间（形成一个 Y 字型链表）。求这俩链表共同后缀的起始节点。

**【解题思路：对齐链表尾部】**

由于后缀共享，意味着一旦两链表相交，后面的节点全是一模一样的。所以相交的节点必定在离表尾等距离的地方。

1. 分别遍历两链表，求出长度 $L_1$ 和 $L_2$。
2. 计算长度差 $d = |L_1 - L_2|$。
3. 让**较长的那个链表**先走 $d$ 步。（让两个链表的剩余长度对齐）。
4. 两个指针一起向后走，比较指针地址（不是比较data！），第一个 `p == q` 的节点就是相交节点。

**【C语言实现】**

```c
LNode* find_addr(LinkList str1, LinkList str2) {
    int len1 = Length(str1), len2 = Length(str2);
    LNode *p = str1->next, *q = str2->next;
    
    // 让较长的表先走差值步
    while (len1 > len2) { p = p->next; len1--; }
    while (len2 > len1) { q = q->next; len2--; }
    
    // 一起走，找交点
    while (p != NULL && p != q) {
        p = p->next;
        q = q->next;
    }
    return p; // 如果没交点，最终 p 和 q 都是 NULL，返回 NULL 正确
}
```

### 真题 3：保留绝对值相等的第一次出现的节点 (2015年 408统考题)

**【题目背景】** 课后综合题第19题。单链表存储的整数绝对值 $\le n$。删除链表中绝对值相等的后续节点，仅保留第一次出现的。

**【解题思路：哈希数组/空间换时间】**

题目给了极其重要的条件：绝对值 $\le n$！这在暗示我们用数组做标记。

1. 申请一个大小为 $n+1$ 的辅助数组 `hash[]`，全部初始化为 0。
2. 遍历链表，取出当前节点的绝对值 $val = |p \rightarrow data|$。
3. 如果 `hash[val] == 0`，说明第一次出现，将 `hash[val] = 1`，指针后移。
4. 如果 `hash[val] == 1`，说明之前出现过了，执行链表删除节点操作。

**【C语言实现】**

```c
#include <stdlib.h>
#include <math.h> // 需要用 abs() 函数

void Delete_Absolute_Duplicates(LinkList head, int n) {
    LNode *p = head, *q;
    // 动态分配标记数组，初始化为0
    int *hash = (int *)calloc(n + 1, sizeof(int)); 
    
    while (p->next != NULL) {
        int val = abs(p->next->data);
        if (hash[val] == 0) {
            hash[val] = 1; // 标记出现过
            p = p->next;   // 保留该节点，指针后移
        } else {
            // 删除 p->next 这个节点
            q = p->next;
            p->next = q->next;
            free(q);
        }
    }
    free(hash); // 别忘了释放辅助空间
}
```

- **复杂度分析：** 遍历一遍链表，时间复杂度 $O(m)$。使用了一个大小为 $n+1$ 的数组，空间复杂度 $O(n)$。

### 真题 4：重排链表 (2019年 408统考题)

**【题目背景】** 课后综合题第20题。设线性表 $L = (a_1, a_2, ..., a_{n-1}, a_n)$，将其重排为 $L' = (a_1, a_n, a_2, a_{n-1}, a_3, ...)$。要求空间复杂度 $O(1)$。

**【解题思路：找中点 + 逆置 + 合并 (堪称考研链表集大成之作)】**

观察重排规律，其实就是把**前半段**和**翻转后的后半段**穿插在一起！

1. **找中点：** 使用快慢指针（`slow` 走一步，`fast` 走两步）。当 `fast` 走到链表尾部时，`slow` 刚好指向中点。将链表从中点断开为 $L_1$ 和 $L_2$。
2. **逆置后半段：** 将 $L_2$ 使用**头插法**重新构造，实现原地翻转（这就是为什么前面说头插法非常重要！）。
3. **交替合并：** 依次从 $L_1$ 和逆置后的 $L_2$ 中摘下节点，交替链接。

**【C语言实现】**

```c
void Rearrange(LinkList &L) {
    if (L->next == NULL || L->next->next == NULL) return; // 空表或只有一个节点
    
    // 1. 找中点，快慢指针
    LNode *slow = L->next, *fast = L->next;
    LNode *pre = L; // 记录slow的前驱
    while (fast != NULL && fast->next != NULL) {
        pre = slow;
        slow = slow->next;
        fast = fast->next->next;
    }
    
    // 2. 断开链表，slow是后半段的第一个节点
    pre->next = NULL;
    
    // 3. 逆置后半段（头插法）
    LNode *L2 = (LNode *)malloc(sizeof(LNode));
    L2->next = NULL;
    LNode *p = slow;
    while (p != NULL) {
        LNode *temp = p->next;
        p->next = L2->next;
        L2->next = p;
        p = temp;
    }
    
    // 4. 交替合并
    LNode *p1 = L->next, *p2 = L2->next;
    LNode *r = L; // 尾指针
    while (p1 != NULL && p2 != NULL) {
        // 取p1
        r->next = p1;
        r = p1;
        p1 = p1->next;
        // 取p2
        r->next = p2;
        r = p2;
        p2 = p2->next;
    }
    // 连接剩余部分
    if (p1 != NULL) r->next = p1;
    if (p2 != NULL) r->next = p2;
    
    free(L2); // 释放临时头结点
}
```
