## 引言

在数据结构的世界里，如果说线性表是自由散漫的队伍，那么**栈（Stack）**和**队列（Queue）**就是被严格约束的特种兵。今天我们要系统学习的是本章的重头戏——**栈**。

无论是计算机的函数调用、浏览器的"后退"功能，还是考研中最常考的括号匹配、表达式求值，都离不开栈。本文将带你从零基础彻底吃透栈的基本概念、顺序实现、链式实现以及共享栈的技巧！

---

## 一、栈的基本概念（Stack）

### 1. 什么是栈？

栈是一种**操作受限的线性表**。它只允许在表的一端进行插入和删除操作。

你可以把它想象成**叠盘子**或者**羽毛球筒**：最后放上去的盘子，总是最先被拿下来；最先放进去的羽毛球，总是压在最底下最后才能拿出来。

- **栈顶（Top）**：允许进行插入和删除操作的一端（最上面）。
- **栈底（Bottom）**：固定的，不允许进行插入和删除操作的一端（最底下）。
- **空栈**：不含有任何数据元素的栈。

### 2. 栈的特性：LIFO

栈的逻辑结构属于线性结构，但它的核心特性是：**后进先出（Last In First Out, LIFO）**。

### 3. 补充考点：合法出栈顺序的数量（卡特兰数）

**【必考知识点】** 如果有 $n$ 个不同的元素进栈，出栈元素的排列个数有多少种？

这里有一个著名的数学公式——**卡特兰数（Catalan Number）**：

$$ \frac{1}{n+1} C_{2n}^n $$

> 考研常出选择题，如果 $n$ 不大（如 $n=3, 4, 5$），建议直接动手模拟；如果直接问数量，套用此公式即可。

## 二、栈的基本操作

不管是哪种方式实现栈，我们都需要提供以下基本操作：

- `InitStack(&S)`：**初始化栈**。构造一个空栈，分配内存空间。
- `DestroyStack(&S)`：**销毁栈**。销毁并释放栈S所占用的内存空间。
- `Push(&S, x)`：**进栈/压栈**。若栈未满，则将 $x$ 加入使其成为新栈顶。
- `Pop(&S, &x)`：**出栈/弹栈**。若栈非空，则弹出栈顶元素，并用 $x$ 返回。
- `GetTop(S, &x)`：**读栈顶元素**。若栈非空，则用 $x$ 返回栈顶元素，**但不出栈**。
- `StackEmpty(S)`：**判空**。判断一个栈是否为空。

## 三、顺序栈的实现（数组实现）

顺序栈利用**一组地址连续的存储单元**存放自栈底到栈顶的数据元素，同时附设一个指针（`top`）指示当前栈顶元素的位置。

### 1. 顺序栈的定义（基础版：`top = -1`）

课件中推荐的第一种方式：初始化时 `top` 指针指向 `-1`。**`top` 始终指向当前栈顶元素的真实位置。**

```c
#define MaxSize 10  // 定义栈中元素的最大个数
typedef struct {
    ElemType data[MaxSize]; // 静态数组存放栈中元素
    int top;                // 栈顶指针
} SqStack;
```

### 2. 核心操作完整代码

#### ① 初始化与判空

```c
// 初始化栈
void InitStack(SqStack &S) {
    S.top = -1;  // 初始化栈顶指针为 -1
}

// 判断栈是否为空
bool StackEmpty(SqStack S) {
    return S.top == -1;  // 栈空返回true，否则返回false
}
```

#### ② 进栈（Push）

```c
bool Push(SqStack &S, ElemType x) {
    if (S.top == MaxSize - 1) { // 栈满，报错
        return false;
    }
    S.top = S.top + 1;  // 指针先加1
    S.data[S.top] = x;  // 新元素入栈
    // 以上两句可等价简写为：S.data[++S.top] = x;
    return true;
}
```

#### ③ 出栈（Pop）与读栈顶元素（GetTop）

```c
// 出栈（删除并返回）
bool Pop(SqStack &S, ElemType &x) {
    if (S.top == -1) {  // 栈空，报错
        return false;
    }
    x = S.data[S.top];  // 栈顶元素先出栈
    S.top = S.top - 1;  // 指针再减1
    // 以上两句可等价简写为：x = S.data[S.top--];
    return true;
}

// 读栈顶元素（只返回不删除）
bool GetTop(SqStack S, ElemType &x) {
    if (S.top == -1) {
        return false;
    }
    x = S.data[S.top]; // x记录栈顶元素
    return true;
}
```

### 3. 【拓展】另一种初始化方式：`top = 0`

如果初始化时设 `S.top = 0`，代表 **`top` 始终指向下一个可以插入的空位置**。

- **入栈**：`S.data[S.top++] = x;` （先赋值，后 top 加 1）
- **出栈**：`x = S.data[--S.top];` （先 top 减 1，后取值）
- **栈满条件**：`S.top == MaxSize`
- **判空条件**：`S.top == 0`

> 考研中这两套逻辑都可能出现，遇到题目时先看清楚 `top` 是初始为 -1 还是 0 即可应对自如！

## 四、共享栈的实现

如果我们要用到两个栈，各自申请一个数组往往会造成"一个栈满了，另一个栈还空着"的空间浪费。于是有了**共享栈**：两个栈共享同一片数组空间，两个栈从数组的两头向中间靠拢。

```c
#define MaxSize 10
typedef struct {
    ElemType data[MaxSize];
    int top0;  // 0号栈栈顶指针
    int top1;  // 1号栈栈顶指针
} ShStack;

// 初始化
void InitStack(ShStack &S) {
    S.top0 = -1;       // 0号栈从下往上增长
    S.top1 = MaxSize;  // 1号栈从上往下增长
}
```

- **0号栈进栈**：`S.data[++S.top0] = x;`
- **1号栈进栈**：`S.data[--S.top1] = x;`
- **核心考点：共享栈满的条件是？**
  **`S.top0 + 1 == S.top1`** （两个指针相邻时，说明栈满了！）

## 五、链栈的实现（链表实现）

顺序栈有空间大小的限制（虽然可以动态分配，但扩容成本高），而**链栈（Linked Stack）**完美解决了这个问题。

**逻辑穿越**：链栈实际上就是一个**只允许在头部进行插入和删除操作的单链表**！

- 进栈：相当于单链表的**头插法**。
- 出栈：相当于单链表的**头删法**。

### 1. 链栈的定义

```c
typedef struct Linknode {
    ElemType data;          // 数据域
    struct Linknode *next;  // 指针域
} *LiStack;                 // 栈类型定义
```

### 2. 代码实现（以不带头结点为例）

由于栈的所有操作都在栈顶（链表头部）进行，**不带头结点**的链表在这里操作更为便捷：栈顶指针 `S` 直接指向第一个存储数据的结点。

#### ① 初始化与进栈

```c
// 初始化不带头结点的链栈
void InitStack(LiStack &S) {
    S = NULL; // 空栈
}

// 进栈（单链表头插法）
bool Push(LiStack &S, ElemType x) {
    struct Linknode* p = (struct Linknode*)malloc(sizeof(struct Linknode));
    if (p == NULL) return false; // 内存分配失败
    p->data = x;
    p->next = S; // 新结点的 next 指向原来的栈顶
    S = p;       // 更新栈顶指针
    return true;
}
```

#### ② 出栈

```c
// 出栈（单链表头删法）
bool Pop(LiStack &S, ElemType &x) {
    if (S == NULL) return false; // 栈空
    struct Linknode* p = S;      // 暂存原栈顶结点
    x = p->data;                 // 记录弹出的值
    S = S->next;                 // 栈顶指针后移
    free(p);                     // 释放结点的内存空间
    return true;
}
```

## 六、经典考点与课后习题解析

### 习题 1：合法出栈序列的判断

**【题目】** 一个栈的入栈序列是 `a, b, c, d, e`，则栈的不可能的输出序列是（ ）。

A. `e, d, c, b, a`
B. `d, e, c, b, a`
C. `d, c, e, a, b`
D. `a, b, c, d, e`

**【解析】** 答案选 **C**。

做这类题目的通用法则是：**在弹出一个元素时，它之前入栈且还没弹出的元素，必须按照逆序排列！**

- A：全进，然后全出，合法。
- B：a,b,c,d进 → d出 → e进 → e出 → c出 → b出 → a出，合法。
- **C错误分析**：d 率先出栈，说明 a, b, c 都已经压入栈中，且都在 d 的底下。d 出来之后，接着 c 出来，合法。再接着 e 进栈 → e 出栈，合法。但此时栈里的元素从上到下应该是 `b, a`。所以接下来的出栈顺序只能是 `b, a`，不可能出现 `a, b`！故 C 序列不合法。

---

### 习题 2：共享栈的指针状态

**【题目】** 设有一个共享栈 `S[0...n-1]`，其中第一个栈栈底在位置 0，第二个栈栈底在位置 n-1。则栈满的条件是（ ）。

A. `top[0] == top[1]`
B. `top[0] == top[1] - 1`
C. `top[0] + 1 == top[1] - 1`
D. `top[0] == n-1`

**【解析】** 答案选 **B**。

本题考察共享栈满的条件判定。第一个栈的指针 `top[0]` 从 0 开始往上增加，第二个栈的指针 `top[1]` 从 n-1 开始往下减少。当两个指针相遇，即它们只相差 1 时，中间不再有空余空间。也就是 `top[0] + 1 == top[1]`，移项得 `top[0] == top[1] - 1`。

---

### 习题 3：链栈的理解

**【题目】** 若用单链表来实现一个栈（不设头结点），并保留 `head` 指针指向栈顶，下面关于出栈和入栈时间的说法，正确的是（ ）。

A. 入栈和出栈的时间复杂度都是 $O(1)$
B. 入栈时间是 $O(n)$，出栈时间是 $O(1)$
C. 入栈时间是 $O(1)$，出栈时间是 $O(n)$
D. 必须遍历到尾部才能操作，都是 $O(n)$

**【解析】** 答案选 **A**。

这是考研非常喜欢考的概念结合题。因为栈的操作被严格限制在一端（栈顶），所以我们把单链表的"头"作为"栈顶"。入栈就是"头插法"，出栈就是删除第一个结点（"头删法"），这两种操作都不需要遍历整个链表，因此时间复杂度均为 $O(1)$。
