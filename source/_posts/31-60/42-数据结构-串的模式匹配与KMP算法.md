## 一、什么是字符串的模式匹配

给定：

- **主串** `S`
- **模式串** `T`

字符串模式匹配要解决的问题是：

> 在主串 `S` 中找到一个与模式串 `T` 值相同的子串，并返回它第一次出现的位置；如果不存在，则返回 `0`。

例如：

- 主串：`“嘿嘿嘿红红火火恍恍惚惚嗨皮开森猴开森笑出猪叫哈哈哈哈”`
- 模式串：`“笑出猪叫”`

若能找到，就返回该模式串在主串中的起始位置。

### 1. 模式串和子串不是一回事

这是 PPT 专门强调的一点。

- **子串**：一定是主串的一部分，所以一定存在于主串中
- **模式串**：只是你拿来做匹配的目标，**不一定**能在主串中找到

例如：

- `“笑出猪叫”` 可能是主串的子串
- `“笑出喵叫”` 可以作为模式串，但它未必是主串的子串

所以模式匹配不是“找一个子串”，而是“判断模式串是否在主串中出现”。

## 二、朴素模式匹配算法

### 1. 核心思想

设：

- 主串长度为 `n`
- 模式串长度为 `m`

主串中一共有 `n - m + 1` 个长度为 `m` 的候选子串。

朴素模式匹配（又叫 BF 算法）的思路非常直接：

1. 取主串中第 1 个长度为 `m` 的子串与模式串比较
2. 若不匹配，就比较下一个长度为 `m` 的子串
3. 直到找到完全匹配的子串，或所有候选子串都试完

### 2. 指针版思路

教材更常用两个指针：

- `i`：指向主串当前正在比较的字符
- `j`：指向模式串当前正在比较的字符

若 `S[i] == T[j]`，则两者同时后移；
若失配，则：

```cpp
i = i - j + 2;
j = 1;
```

这条式子的含义是：

- 当前比较失败时，主串应回到“**当前候选子串的下一个起点**”
- 模式串重新回到第 `1` 个字符开始匹配

### 3. 为什么是 `i = i - j + 2`

这是很多人死记硬背的地方，这里补一下推导。

假设失配前：

- 当前候选子串起点是 `k`
- 已经匹配到了模式串第 `j` 个字符

由于此前匹配了 `j - 1` 个字符，所以此时：

$$i = k + (j - 1)$$

于是原候选子串起点：

$$k = i - j + 1$$

下一个候选子串应从 `k + 1` 开始，因此新的主串位置：

$$i = k + 1 = i - j + 2$$

这就是这个公式的来源。

### 4. 代码实现

```cpp
#include <iostream>
using namespace std;

#define MAXLEN 255

typedef struct {
    char ch[MAXLEN + 1];
    int length;
} SString;

int Index_BF(SString S, SString T) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (S.ch[i] == T.ch[j]) {
            i++;
            j++;
        } else {
            i = i - j + 2;
            j = 1;
        }
    }
    if (j > T.length) return i - T.length;
    return 0;
}
```

### 5. 时间复杂度

最坏情况下：每个候选子串都要比较 `m` 个字符，一共有 `n - m + 1` 个候选子串，因此：

$$O((n-m+1)m) = O(nm)$$

最典型的最坏情况是：

- 主串：`aaaaaaa...aaab`
- 模式串：`aaaaab`

前面每次都几乎匹配成功，只在最后一个字符失败，于是产生大量重复比较。

## 三、从朴素匹配到 KMP：优化思路

### 1. 朴素算法浪费在哪里

设模式串 `T = "abaabc"`。

如果已经成功匹配到：

```text
主串：a b a a b ?
模式：a b a a b c
```

说明在失配字符之前，主串当前这 5 个字符**已经和模式串前 5 个字符完全相同**。

这意味着：

- 已经比较过的那部分信息不是“没用了”
- 某些候选子串其实根本没必要再试

### 2. KMP 的核心直觉

**主串指针不回溯，只有模式串指针回跳。**

因为主串中已经比较过的字符信息还在：

- 不匹配字符之前，主串前缀和模式串前缀一定相等
- 真正需要做的是：找到模式串内部一个“还能继续接上”的位置

也就是：

> 让模式串向右滑动，使得“已经匹配成功的那一段”的某个前缀，重新和它的某个后缀对齐。

这正是 `next` 数组要记录的东西。

## 四、KMP 算法与 `next` 数组

### 1. `next` 数组的含义

在王道教材的 1 基下标写法里：

- 当模式串第 `j` 个字符失配时
- 模式串指针应跳到 `next[j]` 位置继续比较

特别地：

- `next[1] = 0`

因为第 1 个字符就失配时，模式串已经没有可复用的信息了，只能比较主串中的下一个相邻子串。

### 2. `next[j]` 的本质

对于 `j > 1`，`next[j]` 可以理解为：

> 在 `T[1...j-1]` 中，最长相等“真前缀 = 真后缀”的长度，再加 1。

为什么要加 1？

因为我们记录的不是“长度”，而是**下一次应该从模式串第几个字符继续比**。

例如 `T = "abaabc"`：

- `j = 6` 失配时，已经匹配的是 `T[1...5] = "abaab"`
- 它的最长相等真前后缀是 `"ab"`，长度为 `2`
- 所以下一步应从模式串第 `3` 个字符继续比较
- 因此 `next[6] = 3`

### 3. `abaabc` 的 `next` 数组

PPT 中给出的结果是：

```text
位置j    1 2 3 4 5 6
模式串    a b a a b c
next[j]  0 1 1 2 2 3
```

这对应的含义分别是：

- 第 1 个字符失配：只能试下一个相邻子串
- 第 2 个字符失配：回到第 1 个字符比较
- 第 3 个字符失配：仍回到第 1 个字符比较
- 第 4 个字符失配：回到第 2 个字符比较
- 第 5 个字符失配：回到第 2 个字符比较
- 第 6 个字符失配：回到第 3 个字符比较

### 4. KMP 的匹配规则

匹配时分两种情况：

```cpp
if (S.ch[i] == T.ch[j]) {
    i++;
    j++;
} else {
    j = next[j];
}

if (j == 0) {
    i++;
    j++;
}
```

这里的 `j == 0` 表示：

- 模式串已经没有可复用前缀
- 只能让主串后移一位，重新从模式串第 1 个字符开始比

### 5. KMP 代码实现

```cpp
void GetNext(SString T, int next[]) {
    int i = 1, j = 0;
    next[1] = 0;
    while (i < T.length) {
        if (j == 0 || T.ch[i] == T.ch[j]) {
            i++;
            j++;
            next[i] = j;
        } else {
            j = next[j];
        }
    }
}

int Index_KMP(SString S, SString T, int next[]) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (j == 0 || S.ch[i] == T.ch[j]) {
            i++;
            j++;
        } else {
            j = next[j];
        }
    }
    if (j > T.length) return i - T.length;
    return 0;
}
```

### 6. 时间复杂度

- 求 `next` 数组：`O(m)`
- 模式匹配过程：`O(n)`

因此 KMP 总复杂度为：

$$O(m+n)$$

这也是它相对朴素算法的根本优势。

## 五、`next` 数组怎么手算

这一节是 PPT 里图示最多、也是很多同学最容易“看懂动画但不会自己算”的地方。

### 1. 永远先写两项

对于王道这套写法：

- `next[1] = 0`
- `next[2] = 1`

原因：

- 第 1 个字符失配，模式串无处可退，只能试下一个相邻子串
- 第 2 个字符失配时，最多只能回到第 1 个字符重新匹配

### 2. 之后怎么推

求 `next[j]` 时，考察的是失配前已经匹配成功的部分 `T[1...j-1]`。

具体方法可以理解为：

1. 在 `j` 前面画一条分界线
2. 寻找 `T[1...j-1]` 的最长相等真前缀和真后缀
3. 若这个公共部分长度为 `k`，则 `next[j] = k + 1`
4. 若不存在，则 `next[j] = 1`

### 3. `google` 例子

PPT 用 `T = "google"` 做了手算练习。按这一套规则，结果是：

```text
位置j    1 2 3 4 5 6
模式串    g o o g l e
next[j]  0 1 1 1 2 1
```

你可以逐个验证：

- `next[5] = 2`，因为 `T[1...4] = "goog"` 的最长相等真前后缀是 `"g"`
- 长度为 `1`，所以下一步从第 `2` 个字符继续比较

### 4. 一个实用记忆法

如果你考试时突然忘了公式，不要硬背，直接：

- 先写出当前失配前已匹配成功的模式串前缀
- 再找它的最长相等真前缀和真后缀
- 公共部分长度 `+1`，就是 `next[j]`

这比死记结论更稳。

## 六、`next` 数组的进一步优化：`nextval`

### 1. 为什么 `next` 还可以优化

看 `T = "abaabc"`，其 `next` 为：

```text
next = [0, 1, 1, 2, 2, 3]
```

问题在于：某些情况下，`j = next[j]` 跳过去之后，**还会立刻再次失配**。

原因是：

- 你已经知道当前主串字符不等于 `T[j]`
- 如果 `T[j] == T[next[j]]`
- 那么跳到 `next[j]` 再比较一次也必然失败

这一步比较其实是浪费的。

### 2. `nextval` 的思想

如果发现：

$$T[j] = T[next[j]]$$

那么就没必要先跳到 `next[j]`，而是可以继续跳到更远的位置。

于是有了优化规则：

```cpp
if (T.ch[j] == T.ch[next[j]])
    nextval[j] = nextval[next[j]];
else
    nextval[j] = next[j];
```

### 3. `abaabc` 的 `nextval`

PPT 给出的结果：

```text
位置j       1 2 3 4 5 6
模式串       a b a a b c
next[j]     0 1 1 2 2 3
nextval[j]  0 1 0 2 1 3
```

与 `next` 相比，`nextval` 跳得更激进，省掉了那些“跳过去也一定失败”的无效比较。

### 4. `nextval` 的两个练习结果

PPT 还给了两个典型手算题：

#### 练习 1：`T = "ababaa"`

```text
next     = [0, 1, 1, 2, 3, 4]
nextval  = [0, 1, 0, 1, 0, 4]
```

#### 练习 2：`T = "aaaab"`

```text
next     = [0, 1, 2, 3, 4]
nextval  = [0, 0, 0, 0, 4]
```

第二个例子最能体现优化意义：前面都是连续相同字符 `a`，如果不用 `nextval`，会反复跳到一堆一定继续失配的位置。

### 5. `nextval` 版代码

```cpp
void GetNextVal(SString T, int nextval[]) {
    int i = 1, j = 0;
    nextval[1] = 0;
    while (i < T.length) {
        if (j == 0 || T.ch[i] == T.ch[j]) {
            i++;
            j++;
            if (T.ch[i] != T.ch[j]) nextval[i] = j;
            else nextval[i] = nextval[j];
        } else {
            j = nextval[j];
        }
    }
}
```

> 不同教材对 `GetNextVal` 的实现细节会有小差异，但核心思想完全一致：跳过必然再次失配的位置。

## 七、KMP 为什么主串指针不回溯

这是本章最核心的理解点，必须单独拎出来。

假设失配时主串指针在 `S[i]`，模式串指针在 `T[j]`，并且此前 `T[1...j-1]` 已匹配成功。

这意味着：

$$S[i-j+1...i-1] = T[1...j-1]$$

也就是说，主串这一段的内容我们**已经完全知道**。

所以接下来要做的不是“重新检查主串过去的字符”，而是：

- 找到模式串内部可复用的前缀
- 让它与刚才已经匹配成功的后缀对齐

因此：

- 主串指针 `i` 不回退
- 模式串指针 `j` 通过 `next` 或 `nextval` 回跳

这就是 KMP 与朴素算法的本质区别。

## 八、做题防坑指南

### 1. 先看下标体系

王道教材通常：

- 串位序从 `1` 开始
- `next[1] = 0`

如果换成 0 基数组写法，公式和代码会跟着变化，千万别混用。

### 2. 分清三个量

- `j`：当前正在比较的模式串位置
- `next[j]`：失配后模式串应该跳到的位置
- `nextval[j]`：进一步跳过必然失配位置后的结果

### 3. `next[j]` 不是“公共前后缀长度”本身

在王道写法里，`next[j]` 记录的是**下一次继续比较的模式串位置**，它等于“最长相等真前后缀长度 + 1”。

### 4. `j == 0` 的含义别忘了

一旦 `j == 0`，说明模式串已经退无可退，只能：

```cpp
i++;
j++;
```

也就是转去尝试主串的下一个相邻子串。

## 总结

把这一节压缩成一句话：

> KMP 的本质，就是利用模式串自身的结构信息，避免主串回溯和重复比较。

你最后应该牢牢记住：

1. **朴素匹配**：失配后主串回到下一个候选子串起点，复杂度 `O(nm)`
2. **KMP**：失配后主串不回溯，模式串按 `next` 回跳，总复杂度 `O(m+n)`
3. **`next` 数组**：记录失配后模式串该跳到哪里继续比较
4. **`nextval` 数组**：进一步跳过那些“跳过去也一定继续失配”的位置

如果你能自己手推出 `abaabc`、`google`、`ababaa`、`aaaab` 这些例子的 `next / nextval`，这章最难的部分就已经过关了。
