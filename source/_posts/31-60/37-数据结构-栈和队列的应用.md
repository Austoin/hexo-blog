## 一、栈的应用：括号匹配

### 1. 核心思想

在我们编写代码或数学公式时，括号总是成对出现的：`()`、`[]`、`{}`。

**规律**：最后出现的左括号，必须最先被右括号匹配。这完美契合了栈的**"后进先出（LIFO）"**特性！

每遇到一个右括号，就去"消耗"一个左括号。

### 2. 匹配失败的三种情况

在从左到右扫描括号序列时，如果出现以下三种情况之一，则匹配失败：

1. **左括号单身**：所有字符扫描完毕后，栈非空（说明有左括号没找到属于它的右括号）
2. **右括号单身**：扫描到一个右括号时，发现栈竟然是空的（说明没有多余的左括号来匹配它）
3. **左右括号不配对**：扫描到一个右括号，弹出的栈顶左括号和它不是同一种类型（比如 `[` 遇到了 `)`）

### 3. 完整 C/C++ 代码实现

```cpp
#include <iostream>
#include <stdbool.h>
using namespace std;

#define MaxSize 10

// 定义顺序栈
typedef struct {
    char data[MaxSize];
    int top;
} SqStack;

// 初始化栈
void InitStack(SqStack &S) { 
    S.top = -1; 
}

// 判空
bool StackEmpty(SqStack S) { 
    return S.top == -1; 
}

// 入栈
bool Push(SqStack &S, char x) {
    if (S.top == MaxSize - 1) return false;
    S.data[++S.top] = x;
    return true;
}

// 出栈
bool Pop(SqStack &S, char &x) {
    if (S.top == -1) return false;
    x = S.data[S.top--];
    return true;
}

// 核心：括号匹配算法
bool bracketCheck(char str[], int length) {
    SqStack S;
    InitStack(S);
    
    for (int i = 0; i < length; i++) {
        // 1. 扫描到左括号，入栈
        if (str[i] == '(' || str[i] == '[' || str[i] == '{') {
            Push(S, str[i]);
        } 
        // 2. 扫描到右括号
        else {
            if (StackEmpty(S)) return false; // 匹配失败情况2：右括号单身
            
            char topElem;
            Pop(S, topElem); // 弹出栈顶元素
            
            // 匹配失败情况3：左右括号不配对
            if (str[i] == ')' && topElem != '(') return false;
            if (str[i] == ']' && topElem != '[') return false;
            if (str[i] == '}' && topElem != '{') return false;
        }
    }
    
    // 匹配失败情况1：左括号单身（全部扫描完，栈还不空）
    return StackEmpty(S); 
}

int main() {
    char str[] = "{[()]}";
    if (bracketCheck(str, 6)) 
        cout << "匹配成功！" << endl;
    else 
        cout << "匹配失败！" << endl;
    return 0;
}
```

## 二、栈的应用：表达式求值

表达式由三个部分组成：**操作数、运算符、界限符（括号）**。

我们平时熟悉的是**中缀表达式**（运算符在两个操作数中间），但计算机很难处理界限符和运算优先级。为了让计算机算得快，波兰数学家发明了**后缀表达式（逆波兰式）**和**前缀表达式（波兰式）**。

### 1. 三种表达式的转换（手算篇）

**原则（考研必记）**："左优先"原则（转后缀）/"右优先"原则（转前缀）。

目的：为了保证我们手算出的结果和计算机机算出的结果完全一致！

**中缀转后缀（左优先）步骤：**

例如中缀表达式：`A + B * (C - D) - E / F`

1. 确定运算顺序：先算 `(C-D)`，再乘 `B`，再除 `E/F`，最后算左边的 `+` 和右边的 `-`
2. 按照 `[左操作数 右操作数 运算符]` 的格式合并：
   - `(C-D)` → `C D -`
   - `B * (C D -)` → `B C D - *`
   - `E / F` → `E F /`
   - `A + (B C D - *)` → `A B C D - * +`
   - 最后一步相减 → **`A B C D - * + E F / -`**

### 2. 后缀表达式的计算（机算篇/用栈实现）

**算法步骤：**

1. 从左往右扫描下一个元素
2. 遇到**操作数**，直接压入栈
3. 遇到**运算符**，从栈中连续**弹出两个操作数**，执行运算，将**运算结果压回栈顶**
4. 最终栈底剩下的唯一一个元素，就是最终结果

**高危避坑警告：**

减法和除法是有先后顺序的！**先出栈的是"右操作数"，后出栈的是"左操作数"！**

比如弹出了 `D` 和 `C`，遇到 `-` 号，计算的应该是 `C - D`，千万别搞反！

**完整代码实现：**

```cpp
#include <iostream>
#include <cstring>
using namespace std;

#define MaxSize 50

// 操作数栈
typedef struct {
    int data[MaxSize];
    int top;
} OpStack;

void InitStack(OpStack &S) { S.top = -1; }
bool StackEmpty(OpStack S) { return S.top == -1; }
bool Push(OpStack &S, int x) {
    if (S.top == MaxSize - 1) return false;
    S.data[++S.top] = x;
    return true;
}
bool Pop(OpStack &S, int &x) {
    if (S.top == -1) return false;
    x = S.data[S.top--];
    return true;
}

// 后缀表达式求值
int EvaluatePostfix(char postfix[], int length) {
    OpStack S;
    InitStack(S);
    
    for (int i = 0; i < length; i++) {
        // 如果是操作数（这里简化为单个数字字符）
        if (postfix[i] >= '0' && postfix[i] <= '9') {
            Push(S, postfix[i] - '0'); // 字符转数字
        }
        // 如果是运算符
        else if (postfix[i] == '+' || postfix[i] == '-' || 
                 postfix[i] == '*' || postfix[i] == '/') {
            int right, left;
            Pop(S, right); // 先出栈的是右操作数
            Pop(S, left);  // 后出栈的是左操作数
            
            int result;
            switch(postfix[i]) {
                case '+': result = left + right; break;
                case '-': result = left - right; break;
                case '*': result = left * right; break;
                case '/': result = left / right; break;
            }
            Push(S, result);
        }
    }
    
    int finalResult;
    Pop(S, finalResult);
    return finalResult;
}

int main() {
    char postfix[] = "34+52-*6/"; // 对应中缀：(3+4)*(5-2)/6
    int result = EvaluatePostfix(postfix, strlen(postfix));
    cout << "后缀表达式的值为：" << result << endl;
    return 0;
}
```

### 3. 中缀表达式转后缀表达式（机算篇/用栈实现）

初始化一个**运算符栈**。从左到右扫描元素：

1. 遇到**操作数**：直接加入后缀表达式
2. 遇到 `(`：直接入栈
3. 遇到 `)`：依次弹出栈内运算符并加入后缀表达式，直到弹出 `(` 为止（注意 `(` 和 `)` 都不加入后缀表达式）
4. 遇到**运算符**：依次弹出栈中**优先级高于或等于**当前运算符的所有运算符，并加入后缀表达式。碰到 `(` 或栈空则停止。然后把当前运算符入栈
5. 扫描完后，把栈中剩余的运算符依次全部弹出加入后缀表达式

**运算符优先级：**
- `*` 和 `/` 优先级相同，高于 `+` 和 `-`
- `+` 和 `-` 优先级相同
- `(` 优先级最低（在栈内时）

### 4. 中缀表达式的直接计算（双栈法）

不转后缀了，一步到位！

需要两个栈：**操作数栈** 和 **运算符栈**。

逻辑其实就是"中缀转后缀" + "后缀求值"的结合体：

- 遇到操作数压入操作数栈
- 遇到运算符按上述规则处理，**每当从运算符栈弹出一个运算符时，就顺便从操作数栈弹出两个数字进行计算，把结果压回操作数栈**

## 三、栈的应用：函数调用与递归

### 1. 递归的底层原理

递归的核心就是：可以把原始问题转换为**属性相同，但规模较小**的问题（如斐波那契数列、阶乘）。

函数调用背后需要用到一个叫**"函数调用栈（递归工作栈）"**的东西。

每进入一层函数/递归，就需要将以下信息压入栈顶：

1. **调用返回地址**（算完之后回到哪里去）
2. **实参**（传入的参数）
3. **局部变量**

### 2. 递归的优缺点

**优点**：代码逻辑非常简洁，容易让人看懂。

**缺点**：效率低。如果递归层数太多，会导致"栈溢出"（Stack Overflow）；有些算法（如求斐波那契数列）可能包含大量的重复计算。

**解决办法**：我们可以自定义一个栈，将递归算法改造成非递归（迭代）算法。

### 3. 递归实现示例：阶乘

```cpp
// 递归实现阶乘
int factorial(int n) {
    if (n == 0 || n == 1) 
        return 1;
    return n * factorial(n - 1);
}

// 非递归实现阶乘（使用循环）
int factorialIterative(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

## 四、队列的应用

队列的特性是**"先进先出（FIFO）"**，通常用于解决排队、顺序处理的问题。

### 1. 树的层次遍历/图的广度优先遍历（BFS）

探索迷宫时，一层一层往外扩展，就是利用队列实现的。

**BFS基本思路：**

1. 将起始节点入队
2. 当队列不空时：
   - 队头节点出队
   - 访问该节点
   - 将该节点的所有未访问邻接节点入队
3. 重复步骤2，直到队列为空

### 2. 操作系统中的应用

**CPU资源的分配**：

多个进程抢占CPU时，采用 FCFS（先来先服务）策略，将就绪进程排成一个队列。

**打印数据缓冲区**：

缓解主机（速度极快）与打印机（速度慢）之间速度不匹配的问题，将打印任务放入队列中按顺序执行。

## 五、经典考点与课后习题解析

### 习题 1：括号匹配的极限深度

**【题目】** 对于一段仅包含圆括号的字符串，如果它是合法匹配的。已知字符串为 `(()(()))`，在利用栈进行括号匹配时，栈中元素最多时有几个元素？

**【解析】**

模拟入栈出栈过程：

1. `(` 入栈，当前栈：`(`
2. `(` 入栈，当前栈：`(` `(` → 深度为2
3. `)` 出栈，当前栈：`(`
4. `(` 入栈，当前栈：`(` `(`
5. `(` 入栈，当前栈：`(` `(` `(` → 深度为3（此时达到最大深度）
6. 后续连续三个 `)`，全部出栈，最终为空

**答案：栈中元素最多时有3个元素。**

---

### 习题 2：中缀转后缀表达式

**【题目】** 请将中缀表达式 `a + b * c - d / e` 转换为后缀表达式。要求严格遵循"左优先"原则。

**【解析】**

遵循左优先，先找优先级最高的，且最左边的运算符：

1. 先算乘法 `b * c` → 组合为 `b c *`
2. 再算除法 `d / e` → 组合为 `d e /`
3. 此时表达式相当于 `a + (b c *) - (d e /)`
4. 算左边的加法 `a + (b c *)` → 组合为 `a b c * +`
5. 最后算减法 `(a b c * +) - (d e /)` → 组合为 `a b c * + d e / -`

**答案：`a b c * + d e / -`**

---

### 习题 3：后缀表达式的机算过程

**【题目】** 利用栈来计算后缀表达式 `3 4 + 5 2 - * 6 /` 的值，请写出遇到操作符 `/` 时，栈中的元素情况。

**【解析】**

严格按照计算机的思路遍历：

1. `3` 入栈。栈：`[3]`
2. `4` 入栈。栈：`[3, 4]`
3. 遇到 `+`：弹出 4（右）和 3（左），计算 3+4=7，`7`入栈。栈：`[7]`
4. `5` 入栈。栈：`[7, 5]`
5. `2` 入栈。栈：`[7, 5, 2]`
6. 遇到 `-`：弹出 2（右）和 5（左），计算 5-2=3，`3`入栈。栈：`[7, 3]`
7. 遇到 `*`：弹出 3（右）和 7（左），计算 7*3=21，`21`入栈。栈：`[21]`
8. `6` 入栈。栈：`[21, 6]`
9. 遇到 `/`：此时准备出栈计算前，栈里面装的是 `21` 和 `6`

**答案：遇到 `/` 操作符时，栈内元素从栈底到栈顶依次为：`21, 6`。**（最终运算结果为 21/6 = 3）

### 习题 4：递归的空间复杂度分析

**【题目】** 使用递归算法计算阶乘 `factorial(n)` 时，其空间复杂度是多少？为什么？

**【解析】**

计算 `factorial(n)` 需要调用 `factorial(n-1)`，一直调用到 `factorial(1)` 或 `0` 为止。

每递归深入一层，系统都要在内存中开辟一个新的"函数调用栈"用来保存当前的返回地址、参数 `n` 等信息。

总共需要深入 $n$ 层，因此栈的最大深度与 $n$ 成正比。

**答案：空间复杂度为 $O(n)$。由于需要用到隐式的系统工作栈来保存每一层递归的状态，递归层数为 $n$ 层，所以需要 $O(n)$ 的辅助空间。这也是递归容易导致栈溢出的根本原因。**

## 总结

1. **括号匹配**：利用栈的LIFO特性，左括号入栈，右括号出栈匹配。注意三种匹配失败的情况。

2. **表达式求值**：
   - 手算：中缀转后缀遵循"左优先"原则
   - 机算：后缀表达式用栈求值，注意先出栈的是右操作数
   - 中缀转后缀：用运算符栈，注意优先级比较

3. **递归**：本质是利用系统栈保存函数调用信息。优点是代码简洁，缺点是效率低、易栈溢出。

4. **队列应用**：主要用于BFS、进程调度、打印缓冲等需要按顺序处理的场景。
