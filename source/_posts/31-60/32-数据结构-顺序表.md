## 什么是顺序表

### 核心定义

**顺序表**是用**顺序存储**的方式实现线性表

核心思想:把逻辑上相邻的元素存储在**物理位置上也相邻**的存储单元中

> 白话理解:就像食堂打饭排队,大家一个挨着一个站,中间没有空隙。只要知道第一个人在哪,又能一眼看出每个人的体型(数据类型大小),就能立马算出第 $i$ 个人在什么具体位置

### 顺序表的特点

**优点**
1. **随机访问**:可以在 $O(1)$ 的时间内找到第 $i$ 个元素,代码实现就是 `data[i-1]`
2. **存储密度高**:每个节点只存储数据元素本身,不需要像链表那样额外花费空间存指针

**缺点**
1. 扩展容量不方便
2. 插入、删除元素不方便(需要移动大量元素)

## 顺序表的代码实现

在C语言中,实现物理上相邻的最直观方式就是**数组**。根据数组大小是否可以改变,顺序表分为**静态分配**和**动态分配**

### 静态分配

使用静态数组实现,一旦内存分配满,就无法再插入新元素

```c
#define MaxSize 10         // 定义最大长度
typedef struct {
    ElemType data[MaxSize]; // 用静态的"数组"存放数据元素
    int length;            // 顺序表的当前长度
} SqList;                  // 顺序表的类型定义
```

### 动态分配

虽然叫动态分配,但**它依然是顺序表,物理内存依然是连续的**!只是借助了 `malloc` 动态申请了一整片连续的内存空间

```c
#define InitSize 10        // 顺序表的初始长度
typedef struct {
    ElemType *data;        // 指示动态分配数组的指针 (指向连续空间首地址)
    int MaxSize;           // 顺序表的最大容量
    int length;            // 顺序表的当前长度
} SeqList;

// 动态申请空间的C语言核心代码:
// L.data = (ElemType *)malloc(sizeof(ElemType) * InitSize);
```

**考研易错点(动态扩容的本质)**

当动态顺序表存满时,需要扩容。**扩容不是在原数组后面直接加一块**,而是申请一块**全新且更大**的连续空间(比如原本是10,现在申请个20的),然后把旧数据**复制**过去,最后用 `free` 释放旧空间

这也意味着,如果申请扩大 $m$ 个空间失败,说明系统当前没有大小为 $n+m$ 个连续的存储空间,而不是没有 $m$ 个连续空间!

## 顺序表的基本操作

考研复习中,必须烂熟于心的是这三个操作的**代码逻辑边界**和**时间复杂度**

### 插入操作

**目标**:在顺序表 $L$ 的第 $i$ 个**位序**(即下标 $i-1$)处插入新元素 $e$

**核心思路**:把第 $i$ 个及以后的元素全部**向后移动**一位,给新元素腾位置

**注意:移动时要从后往前移,否则会被覆盖!**

```c
bool ListInsert(SqList &L, int i, ElemType e) {
    // 1. 判断 i 的合法性 (可以在第1个到第 length+1 个位置插入)
    if(i < 1 || i > L.length + 1) return false;
    // 2. 判断表是否满了
    if(L.length >= MaxSize) return false;
    
    // 3. 后移元素腾出空位(注意 j 的起始值是 L.length)
    for(int j = L.length; j >= i; j--) {
        L.data[j] = L.data[j-1]; 
    }
    L.data[i-1] = e; // 在位置 i 处放入 e
    L.length++;      // 长度加1
    return true;
}
```

**时间复杂度**
- 最好时间复杂度:插在表尾,不用移动。$O(1)$
- 最坏时间复杂度:插在表头,移动 $n$ 次。$O(n)$
- 平均时间复杂度:$O(n)$

### 删除操作

**目标**:删除顺序表 $L$ 的第 $i$ 个**位序**的元素,并用引用变量 `e` 返回

**核心思路**:把第 $i$ 个之后的元素全部**向前移动**一位,覆盖掉要删除的元素

**注意:移动时要从前往后移!**

```c
bool ListDelete(SqList &L, int i, ElemType &e) {
    // 1. 判断 i 的合法性
    if(i < 1 || i > L.length) return false;
    
    e = L.data[i-1]; // 2. 将被删除的元素赋值给 e
    
    // 3. 前移元素进行覆盖
    for(int j = i; j < L.length; j++) {
        L.data[j-1] = L.data[j];
    }
    L.length--;      // 长度减1
    return true;
}
```

**时间复杂度**
- 最好时间复杂度:删除表尾元素,不用移动。$O(1)$
- 最坏时间复杂度:删除表头元素,移动 $n-1$ 次。$O(n)$
- 平均时间复杂度:$O(n)$

### 查找操作

**按位查找 `GetElem(L, i)`**

直接通过 `L.data[i-1]` 获取。**时间复杂度 $O(1)$**

这正是顺序表"随机存取"特性的体现!

**按值查找 `LocateElem(L, e)`**

需要从头到尾遍历数组,找等于 `e` 的元素。**时间复杂度 $O(n)$**

**C语言结构体比对陷阱**:在纯C语言中,不能直接用 `==` 来比较两个结构体变量是否相等,必须拆解开对比里面的每个分量。但在考研数据结构初试的手写代码中,王道教材指出可以直接使用 `==`,阅卷老师通常算对

## 答案与解析

**Q1. A**

顺序表的核心优点就是"随机存取"和"存储密度大"。因为它不需要额外分配空间来存指针;相反,插入和删除需要移动大量元素,极不方便(BC错)

**Q2. B**

顺序表的存储空间 = 表长(n) × sizeof(单个元素的大小)。显然,它与元素的个数(A)、元素的类型大小(C, D)有关,而跟你把哪个元素放在前面还是后面(元素的存放顺序)毫无关系

**Q3. D**

关键词:**"存取第 i 个元素"**。只要看到极度频繁的按位置访问,首选具有随机存取特性(时间复杂度为 $O(1)$)的顺序表。其它链表都需要从头开始遍历,时间复杂度为 $O(n)$

**Q4. C**

自己举个特例马上明白:假如长度 $n=5$,我要删掉第 $i=5$ 个元素(最后一个),显然不需要移动任何元素,移动个数为 0。代入选项,只有 C ($5-5=0$) 成立

**Q5. C**

A错:顺序表是逻辑结构,一维数组是存储结构(物理结构)。B错:顺序表的绝对特征就是逻辑相邻物理必相邻。D错:线性表要求数据元素必须是相同类型的。C对:数组和顺序表都支持通过下标 $O(1)$ 随机访问

**Q6. C**

"随机存取"指的是"按位置(序号)访问",即找第 $i$ 个元素的时间复杂度是 $O(1)$,不需要像链表那样遍历,因此与元素总个数 $n$ 无关。而找值为 $x$ 的元素(按值查找)需要遍历,时间复杂度是 $O(n)$

**Q7. A**

该表有两个最常用的操作:① 存取任一指定序号元素(顺序表 $O(1)$,链表 $O(n)$);② 在最后进行插入删除(顺序表插在表尾不需要移动元素,时间也是 $O(1)$)。因此顺序表完美契合这两个需求

**Q8. C**

- Ⅰ. 访问第 $i$ 个结点是 data[i-1],前驱是 data[i-2],都是 $O(1)$
- Ⅱ. 在末尾插入无需移动元素,$O(1)$
- Ⅲ. 删第 1 个结点,后面的 $n-1$ 个元素全都要前移,$O(n)$
- Ⅳ. 在第 $i$ 个结点后插入,需要后移 $n-i$ 个元素,平均 $O(n)$

**Q9. C**

- Ⅰ. 顺序表 $O(1)$,链表 $O(n)$,顺序表高
- Ⅱ. 顺序表直接通过下标交换,链表还需要顺着指针寻找 3 个节点,虽然都是常数级别,但顺序表操作明显更快捷
- Ⅲ. 顺序输出所有元素,两者都要遍历一遍,都是 $O(n)$,效率相当

**Q10. C**

访问第 $i$ 个位置直接使用下标,为 $O(1)$;在第 $i$ 个位置插入,需要把 $i$ 到 $n$ 的元素全部后移,平均时间复杂度为 $O(n)$

**Q11. C**

A 排序通常需要 $O(n\log n)$ 或 $O(n^2)$。B 和 D 都会引起大量元素的移动,为 $O(n)$。C 相当于按位查找并重新赋值,data[i-1] = new_value;,时间复杂度为 $O(1)$

**Q12. D**

考查动态顺序表扩容的底层逻辑。动态扩容(如使用 realloc 或 malloc)时,系统需要寻找一块全新且连续的内存空间,大小为旧容量加新容量,即 $n+m$。申请失败是因为找不到 $n+m$ 个连续的空间,而不是仅仅找不到 $m$ 个

**Q13. D**

- A: 按值查找,平均是 $O(n)$,即使是有序表用二分查找也是 $O(\log n)$
- B: 插入操作,需要移动元素,平均 $O(n)$
- C: 删除操作,需要移动元素,平均 $O(n)$
- D: 获取第 i 个元素,即按位查找,利用数组下标直接访问 `data[i-1]`,时间复杂度完美 $O(1)$!

## 核心总结

复习本节时,请闭上眼睛在脑海中回答这三个问题:

1. **顺序表和数组的区别是什么?**

顺序表是逻辑结构+物理结构的结合,它不仅要求物理空间连续,还封装了长度属性和增删改查的操作

2. **手写插入和删除代码时,`for` 循环的方向分别是什么?**

插入:从后往前移;删除:从前往后移

3. **当题目问到"最快访问第 i 个元素"时,你第一反应是什么?**

具有随机访问特性的顺序表
